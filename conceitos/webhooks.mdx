## Visão geral

Webhooks permitem que sua aplicação receba notificações automáticas quando eventos ocorrem, como a autorização ou rejeição de uma nota fiscal.

Configure a URL do webhook no **Painel > Configurações > Webhooks**.

## Eventos

| Evento                | Descrição                                      |
|-----------------------|------------------------------------------------|
| `invoice.authorized`  | Nota fiscal autorizada com sucesso             |
| `invoice.rejected`    | Nota fiscal rejeitada pela prefeitura/SEFAZ    |
| `invoice.cancelled`   | Nota fiscal cancelada com sucesso              |
| `invoice.error`       | Erro no processamento da nota fiscal           |

## Payload

Todas as notificações seguem a mesma estrutura:

```json
{
  "event": "invoice.authorized",
  "timestamp": "2026-04-15T14:35:22Z",
  "data": {
    "invoice_id": "inv_a1b2c3d4e5f6",
    "company_id": "comp_9f8a7b6c5d4e3f2a",
    "status": "authorized",
    "external_id": "pedido-001",
    "numero_nota": "20260001",
    "valor": 5000.00,
    "pdf_url": "https://emitezap.com.br/api/partner/v1/invoices/inv_a1b2c3d4e5f6/pdf",
    "xml_url": "https://emitezap.com.br/api/partner/v1/invoices/inv_a1b2c3d4e5f6/xml",
    "tomador": {
      "cpf_cnpj": "987.654.321-00",
      "razao_social": "Maria Fernanda Silva"
    }
  }
}
```

## Assinatura HMAC-SHA256

Cada notificação inclui o header `X-EmiteZap-Signature` com uma assinatura HMAC-SHA256 do payload, usando o webhook secret configurado no painel.

**Sempre verifique a assinatura** antes de processar o webhook para garantir que a requisição é legítima.

### Verificação em PHP

```php
<?php

$payload = file_get_contents('php://input');
$signature = $_SERVER['HTTP_X_EMITEZAP_SIGNATURE'] ?? '';
$secret = env('EMITEZAP_WEBHOOK_SECRET');

$expected = hash_hmac('sha256', $payload, $secret);

if (!hash_equals($expected, $signature)) {
    http_response_code(401);
    echo json_encode(['error' => 'Assinatura inválida']);
    exit;
}

$event = json_decode($payload, true);

switch ($event['event']) {
    case 'invoice.authorized':
        // Nota autorizada — atualizar pedido, notificar cliente
        break;
    case 'invoice.rejected':
        // Nota rejeitada — verificar dados e reemitir
        break;
    case 'invoice.cancelled':
        // Nota cancelada
        break;
    case 'invoice.error':
        // Erro no processamento
        break;
}

http_response_code(200);
echo json_encode(['received' => true]);
```

### Verificação em Node.js

```javascript
const crypto = require('crypto');

app.post('/webhooks/emitezap', (req, res) => {
  const payload = JSON.stringify(req.body);
  const signature = req.headers['x-emitezap-signature'];
  const secret = process.env.EMITEZAP_WEBHOOK_SECRET;

  const expected = crypto
    .createHmac('sha256', secret)
    .update(payload)
    .digest('hex');

  if (!crypto.timingSafeEqual(
    Buffer.from(expected),
    Buffer.from(signature)
  )) {
    return res.status(401).json({ error: 'Assinatura inválida' });
  }

  const event = req.body;

  switch (event.event) {
    case 'invoice.authorized':
      // Nota autorizada
      break;
    case 'invoice.rejected':
      // Nota rejeitada
      break;
    case 'invoice.cancelled':
      // Nota cancelada
      break;
    case 'invoice.error':
      // Erro no processamento
      break;
  }

  res.status(200).json({ received: true });
});
```

## Política de retentativa

Se o seu servidor não responder com status `2xx`, a EmiteZap fará novas tentativas:

| Tentativa | Intervalo       |
|-----------|-----------------|
| 1ª        | Imediata        |
| 2ª        | Após 1 minuto   |
| 3ª        | Após 5 minutos  |
| 4ª        | Após 30 minutos |

Após 3 retentativas falhas (4 tentativas no total), o webhook é marcado como falho e pode ser consultado no painel.

<Tip>
  Responda rapidamente ao webhook (em menos de 5 segundos) e processe a lógica de negócio de forma assíncrona para evitar timeouts.
</Tip>
